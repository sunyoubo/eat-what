## 2. 技术栈选型

### 2.1 前端技术栈

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 小程序框架 | Taro 3.x | 一套代码多端运行（微信/飞书/H5） |
| UI 组件库 | NutUI / Taro UI | 移动端组件库 |
| 状态管理 | Zustand | 轻量级状态管理 |
| 网络请求 | Axios + React Query | 数据获取与缓存 |
| 图表可视化 | ECharts / F2 | 营养数据可视化 |

### 2.2 后端技术栈

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 主语言 | Node.js (NestJS) | 快速开发，适合 I/O 密集型 |
| 推荐引擎 | Python (FastAPI) | 机器学习模型部署 |
| API 网关 | Kong | 路由、限流、认证 |
| 数据库 | MySQL 8.0 | 核心业务数据 |
| 缓存 | Redis 7.0 | 会话、热点数据 |
| 搜索引擎 | Elasticsearch 8.x | 菜品搜索 |
| 消息队列 | RabbitMQ / Kafka | 异步任务、日志收集 |
| 对象存储 | 阿里云 OSS | 用户头像、菜品图片 |

### 2.3 基础设施

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 容器化 | Docker + Kubernetes | 容器编排 |
| CI/CD | GitHub Actions / Jenkins | 自动化部署 |
| 监控 | Prometheus + Grafana | 指标监控 |
| 日志 | ELK Stack | 日志收集分析 |
| APM | SkyWalking | 链路追踪 |

## 3. 核心模块设计

### 3.1 用户服务 (User Service)

#### 3.1.1 数据库设计

```sql
-- 用户基础信息表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    openid VARCHAR(64) UNIQUE NOT NULL COMMENT '小程序 openid',
    unionid VARCHAR(64) UNIQUE COMMENT '跨应用统一 ID',
    phone VARCHAR(20) COMMENT '手机号',
    nickname VARCHAR(50) COMMENT '昵称',
    avatar_url VARCHAR(255) COMMENT '头像',
    
    -- 身体数据
    gender ENUM('male', 'female') NOT NULL,
    birth_date DATE NOT NULL COMMENT '出生日期',
    height_cm DECIMAL(5,2) NOT NULL COMMENT '身高 cm',
    weight_kg DECIMAL(5,2) NOT NULL COMMENT '体重 kg',
    activity_level ENUM('sedentary', 'light', 'moderate', 'active') DEFAULT 'sedentary',
    health_goal ENUM('maintain', 'lose_weight', 'gain_muscle', 'control_sugar') DEFAULT 'maintain',
    
    -- 计算字段
    bmr DECIMAL(8,2) COMMENT '基础代谢率',
    tdee DECIMAL(8,2) COMMENT '每日总消耗',
    bmi DECIMAL(5,2) COMMENT '身体质量指数',
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_openid (openid),
    INDEX idx_unionid (unionid)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 用户口味偏好表
CREATE TABLE user_preferences (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    
    -- 口味倾向 (1-5 分，5 最喜欢)
    taste_spicy TINYINT DEFAULT 3 COMMENT '辣度偏好 0-5',
    taste_sweet TINYINT DEFAULT 3 COMMENT '甜度偏好 0-5',
    taste_salty TINYINT DEFAULT 3 COMMENT '咸度偏好 0-5',
    taste_light TINYINT DEFAULT 3 COMMENT '清淡偏好 0-5',
    
    -- 忌口食材 (JSON 数组)
    avoid_ingredients JSON COMMENT '["香菜", "葱", "蒜"]',
    
    -- 菜系偏好 (JSON 数组，带权重)
    cuisine_preferences JSON COMMENT '[{"cuisine": "sichuan", "weight": 0.8}]',
    
    -- 过敏原
    allergens JSON COMMENT '["peanut", "seafood"]',
    
    -- 预算范围
    budget_min DECIMAL(8,2) DEFAULT 15,
    budget_max DECIMAL(8,2) DEFAULT 50,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 用户体重记录表
CREATE TABLE user_weight_logs (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    weight_kg DECIMAL(5,2) NOT NULL,
    body_fat_percent DECIMAL(5,2) COMMENT '体脂率',
    measure_time DATETIME NOT NULL,
    source ENUM('manual', 'smart_scale') DEFAULT 'manual',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    INDEX idx_user_time (user_id, measure_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

#### 3.1.2 核心 API

```typescript
// NestJS Controller 示例
@Controller('api/v1/users')
export class UserController {
  
  // 用户注册/登录（飞书 OAuth）
  @Post('auth/feishu')
  async feishuLogin(@Body() dto: FeishuAuthDto) {
    // 1. 验证飞书 code
    // 2. 获取用户信息
    // 3. 创建或更新本地用户
    // 4. 返回 JWT token
  }
  
  // 更新身体数据
  @Patch('profile/body')
  @UseGuards(JwtAuthGuard)
  async updateBodyData(
    @Req() req: Request,
    @Body() dto: BodyDataDto
  ) {
    const userId = req.user.id;
    // 更新身体数据并重新计算 BMR/TDEE/BMI
    return this.userService.updateBodyData(userId, dto);
  }
  
  // 获取用户偏好
  @Get('preferences')
  @UseGuards(JwtAuthGuard)
  async getPreferences(@Req() req: Request) {
    return this.userService.getPreferences(req.user.id);
  }
  
  // 更新偏好
  @Patch('preferences')
  @UseGuards(JwtAuthGuard)
  async updatePreferences(
    @Req() req: Request,
    @Body() dto: PreferenceDto
  ) {
    return this.userService.updatePreferences(req.user.id, dto);
  }
  
  // 记录体重
  @Post('weight')
  @UseGuards(JwtAuthGuard)
  async logWeight(
    @Req() req: Request,
    @Body() dto: WeightLogDto
  ) {
    return this.userService.logWeight(req.user.id, dto);
  }
}
```

## 5. 安全设计

### 5.1 数据安全
- 敏感数据加密存储（AES-256）
- 传输层 TLS 1.3 加密
- 数据库访问白名单
- SQL 注入防护（参数化查询）

### 5.2 认证授权
- JWT Token 有效期 2 小时
- Refresh Token 有效期 7 天
- RBAC 权限控制
- API 限流（每用户 100 次/分钟）

### 5.3 隐私合规
- 用户数据脱敏展示
- 支持 GDPR 数据导出/删除
- 隐私政策明确告知
- 第三方 SDK 合规审查
